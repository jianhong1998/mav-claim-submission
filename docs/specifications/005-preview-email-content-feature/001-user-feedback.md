# Background

Users feedback that they are unsure what will be sent out in the email.

# Requirements

## Preview email content feature

Users are requesting for a preview feature when creating a draft claim.

This preview feature should be showing the actual email content to be sent with the current claim submission data.

# Technical consideration

## Email content

As email template and the content fill-up functions are at the backend, backend should have an endpoint to return the email content (maybe in HTML format?) with the current claim submission data.
